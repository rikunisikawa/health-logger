# deep-research-agent の責務と使い方

## 概要

「調査・分析・意思決定補助」に特化したエージェント。
pm-agent や人間が意思決定する際の「判断材料の収集役」として機能する。

**実装・Issue 操作・設計決定は一切行わない。データで現状を示すことが唯一の責務。**

## いつ使うか

| シーン | 使い方 |
|--------|--------|
| 停滞している Issue を把握したい | `stale-detect` 分析を実行 |
| スプリントのベロシティを確認したい | `velocity` 分析を実行 |
| リスクを網羅的に把握したい | `risk-scan` 分析を実行 |
| 技術選定の比較材料が欲しい | `tech-research` 分析を実行 |
| 過去の設計決定の経緯を知りたい | `decision-history` 分析を実行 |
| 重複している Issue を見つけたい | `duplicate-scan` 分析を実行 |
| PM として「次に何を判断すべきか」を整理したい | `pm-summary` 分析を実行 |

## 実行例

```
「deep-research-agent を使って直近2週間の Issue 傾向を調べて」
「停滞している Issue と PR を洗い出して、リスクマトリクスを作って」
「Athena クエリの最適化について技術調査して」
「過去に DynamoDB を選んだ理由を調べて」
```

## 分析メニュー一覧

| 分析名 | 内容 | 所要時間 |
|--------|------|---------|
| `issue-trend` | 直近N日の Issue 作成・クローズ傾向 | ~30秒 |
| `risk-scan` | open Issue/PR のリスク分類 | ~30秒 |
| `velocity` | スプリント別クローズ数推移 | ~30秒 |
| `stale-detect` | 14日以上更新なしの Issue/PR 検出 | ~20秒 |
| `duplicate-scan` | 類似 Issue の重複検出 | ~30秒 |
| `tech-research` | 外部ドキュメントを参照した技術比較 | ~1分 |
| `decision-history` | 過去の設計決定経緯の要約 | ~1分 |
| `pm-summary` | 上記を組み合わせた PM 向けサマリー | ~2分 |

## 出力形式

常にマークダウン形式で出力。意思決定・推奨事項は含めない。
「pm-agent への引き渡し事項」として「次に判断すべきことの候補」を記載する。

## 制約

- ファイルへの書き込みは行わない（stdout のみ）
- 他エージェントへの委譲は行わない
- GitHub 書き込み操作は一切行わない
