このリポジトリの Claude Code 開発環境をレビューし、改善案を docs/claude-code-dev-env-review.md に書き込んでください。

## 手順

### 1. 公式ドキュメントの確認

以下の URL を fetch して最新情報を確認してください。取得できない場合は「参照不可（URL のみ）」と明記してください。

- https://docs.anthropic.com/en/docs/claude-code/settings
- https://docs.anthropic.com/en/docs/claude-code/hooks
- https://docs.anthropic.com/en/docs/claude-code/mcp
- https://docs.anthropic.com/en/docs/claude-code/memory
- https://docs.anthropic.com/en/docs/claude-code/sub-agents

### 2. リポジトリ設定の読み込み

以下をすべて読み込んでください。

- CLAUDE.md
- .claude/settings.json
- .mcp.json
- .claude/agents/ 配下の全ファイル
- .claude/commands/ 配下の全ファイル
- .claude/hooks/ 配下の全ファイル
- .claude/rules/ 配下の全ファイル
- .claude/skills/ 配下の全ファイル

### 3. 改善案の作成と出力

`dev-env-review` スキルのチェックリスト（A〜I）に沿って評価し、以下のフォーマットで `docs/claude-code-dev-env-review.md` に書き込んでください。

```
# Claude Code 開発環境 改善レポート

生成日時: <date コマンドで取得>
生成方法: scripts/run-dev-env-review.sh（非インタラクティブ実行）

## 参照した公式ドキュメント

## 現状サマリー

## 改善案

### A. CLAUDE.md
### B. settings.json
### C. Hooks
### D. MCP サーバー
### E. Agents / Skills / Commands
### F. セキュリティ
### G. コスト最適化
### H. 今後の拡張案

## 優先度マトリクス

| 優先度 | 改善項目 | 工数感 | 理由 |
|--------|---------|--------|------|

## 不明点・確認が必要な事項
```

### 4. コミット

書き込み完了後、以下を実行してください。

```bash
git add docs/claude-code-dev-env-review.md
git commit -m "docs: Claude Code 開発環境の改善レポートを更新 ($(date +%Y-%m-%d))

Generated by scripts/run-dev-env-review.sh (non-interactive).
Co-Authored-By: Claude Code <noreply@anthropic.com>"
```

## 制約

- `docs/claude-code-dev-env-review.md` 以外のファイルは変更しない
- 推測・憶測は書かず、不明な点は「不明」と明記する
- 大規模なリファクタリング提案は「今後の拡張案」に限定する
