### 3. 改善案の作成

以下の観点でレビューし、改善案を作成してください。

#### A. GitHub Actions での Claude Code 実行
- 現在の `.github/workflows/claude-code-review.yml` の構成は適切か
- `anthropics/claude-code-action` の最新の使い方と乖離がないか
- permissions・secrets の設定は最小権限か
- トリガー設定（workflow_dispatch / schedule）の妥当性

#### B. CLAUDE.md の整理
- 記述の重複・矛盾・古くなっている箇所
- GitHub Actions 実行時の注意事項が明記されているか
- 禁止事項・コーディング規約の網羅性

#### C. `.claude/settings.json` の整理
- permissions の allow/deny が適切か（過剰・不足がないか）
- hooks の設定は正しく機能するか
- CI 環境と開発環境で設定を分けるべき箇所があるか

#### D. Hooks の評価
- 各 hook の実装は意図通りに動作するか
- GitHub Actions 環境での動作に問題はないか
- 追加すべき hook・削除すべき hook はあるか

#### E. MCP サーバーの評価
- 各 MCP サーバーの必要性・有効性
- ローカル / GitHub Actions 両環境での動作可否
- セキュリティ上の懸念（`FETCH_ALLOWLIST` の範囲など）

#### F. Agents / Skills / Commands の評価
- 定義されているエージェントの役割分担は適切か
- スキルのカバレッジに漏れはないか
- `/plan`・`/tdd` などのコマンドは現状に合っているか

#### G. セキュリティ
- シークレット・認証情報の扱いに問題はないか
- `deny` リストで保護できていない危険操作はないか
- MCP の権限スコープは適切か

#### H. コスト最適化
- GitHub Actions での実行頻度・モデル選択の妥当性
- 不要なツール呼び出しを減らせる設定はあるか
- `max_turns` の適切な値

#### I. 今後の拡張案
- 追加すると有用な GitHub Actions ワークフロー
- 追加すると有用な agents / skills
- devcontainer との連携可能性
