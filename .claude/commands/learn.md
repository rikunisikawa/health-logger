# /learn — セッションパターン抽出

## 実行手順

### 1. セッションを振り返る

このセッションで解決した問題を特定する:
- バグの根本原因と修正手順
- 非自明なデバッグ手順・ツールの組み合わせ
- ライブラリのクセ・API の挙動
- プロジェクト固有のパターン（AWS サービス連携・Iceberg 制約など）

### 2. 保存価値を判断する

以下に当てはまる場合は保存する:
- 同じミスを繰り返す可能性がある（例: Iceberg スキーマ変更後の ALTER TABLE）
- ドキュメントに書かれていない挙動
- 30 分以上かかったデバッグ

既に `memory/` に記録済みのものは更新のみ行う。

### 3. memory ファイルに保存

保存先: `.claude/projects/.../memory/` または `~/.claude/projects/.../memory/`

**feedback 型**（ミス・修正指示）:
```markdown
---
name: <簡潔な名前>
description: <1行説明>
type: feedback
---

<ルール>

**Why:** <なぜこの失敗が起きたか>
**How to apply:** <次回どう適用するか>
```

**project 型**（プロジェクト固有の知識）:
```markdown
---
name: <簡潔な名前>
description: <1行説明>
type: project
---

<事実・決定事項>

**Why:** <なぜこうなっているか>
**How to apply:** <設計・実装時にどう考慮するか>
```

### 4. MEMORY.md のインデックスを更新

`memory/MEMORY.md` に新しいファイルへのリンクを追加する。

## 保存しないもの

- コードのパターンや規約（コードから読み取れるもの）
- git 履歴やコミット内容
- この会話の一時的なコンテキスト
