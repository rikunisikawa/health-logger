# /risk-review — リスクレビュー

`risk-review` スキルを参照し、以下の手順で実行せよ。

## 手順

1. open Issue の更新日を確認し、14日以上更新のないものを特定する
2. open PR のレビュー状態を確認し、stale なものを特定する
3. 直近のCI実行結果を確認する
4. コード内の TODO/FIXME/セキュリティリスクパターンをスキャンする
5. リスクマトリクスを生成してコンソールに表示する

## 実行例

```
/risk-review
```

## 注意

- リスクの「報告」のみを行う。Issue のクローズ・変更は行わない
- 対処は人間または pm-agent が判断する
